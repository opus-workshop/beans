## Problem

353 raw `println!/eprintln!` calls scattered across all commands. No way to:
- Add consistent coloring/theming
- Support `--quiet` mode globally
- Support `--json` output mode for all commands (not just `bn run`)
- Test output without capturing stdout

## Approach

Create a thin output abstraction. Not a logging framework — just a structured way to emit messages.

### Files to create/modify
- `src/output.rs` — Output struct with methods: info(), success(), warn(), error(), json()
- `src/lib.rs` — add `pub mod output;`
- Gradually migrate commands (don't do all 353 at once — just establish the pattern)

### Design
```rust
pub struct Output {
    json: bool,
    quiet: bool,
}

impl Output {
    pub fn info(&self, msg: &str) { if !self.quiet { eprintln!("{}", msg); } }
    pub fn success(&self, id: &str, msg: &str) { eprintln!("  ✓ {}  {}", id, msg); }
    pub fn warn(&self, msg: &str) { eprintln!("  ⚠ {}", msg); }
    pub fn error(&self, msg: &str) { eprintln!("  ✗ {}", msg); }
}
```

Start by wiring it into 2-3 commands as proof of concept.

### Acceptance Criteria
- [ ] Output type exists in src/output.rs
- [ ] At least 2 commands use it instead of raw println
- [ ] --quiet support works for those commands
- [ ] All tests pass
