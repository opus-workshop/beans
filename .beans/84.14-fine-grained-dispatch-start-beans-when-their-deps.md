## Problem

Current wave model is coarse: wave N+1 waits for ALL of wave N to complete, even if a specific bean's deps finished early. For a parent with:
- Wave 1: A (2min), B (30min)
- Wave 2: C (depends on A only)

C waits 30min for B even though A finished in 2min.

## Approach

Replace wave-based execution with a ready-queue model:
1. Start with all beans that have no deps
2. When any bean finishes, check if new beans are now unblocked
3. Immediately dispatch newly-unblocked beans (up to max_jobs)

### Design
- Keep `compute_waves()` for dry-run display and planning
- Add `dispatch_ready()` that uses a dynamic ready queue for execution
- Track completed set, check requires/dependencies against it
- Same thread-based approach, just more granular scheduling

### Files to modify
- `src/commands/run.rs` — add dynamic dispatch alongside wave execution
