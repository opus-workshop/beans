## Problem

`src/commands/close.rs` is 2,737 lines — a god file doing verification, fail-first logic, archiving, hook execution, parent auto-close, output capture, and history recording.

## Approach

Convert `src/commands/close.rs` into `src/commands/close/` directory module.

### New structure
```
src/commands/close/
├── mod.rs          — cmd_close entry point + orchestration (<800 lines)
├── verify.rs       — verify command execution, fail-first logic, output capture
├── archive.rs      — archive bean to .beans/archive/, auto-close parent
└── history.rs      — run records, attempt tracking, circuit breaker
```

### Approach
1. Create directory and mod.rs
2. Move verify/fail-first logic → verify.rs
3. Move archive + parent auto-close → archive.rs
4. Move RunRecord/history logic → history.rs  
5. Keep cmd_close in mod.rs as the orchestrator
6. Re-export public items from mod.rs

### Acceptance Criteria
- [ ] close/ is a directory module with mod.rs, verify.rs, archive.rs, history.rs
- [ ] mod.rs is under 800 lines
- [ ] All 75 close tests still pass
- [ ] No public API changes (other modules import the same paths)
