## What to implement

Prevent beans that touch the same files from running concurrently. Instead of just warning about file locks, the scheduler should *defer* conflicting beans until the file is free.

## Files

- `src/commands/run/ready_queue.rs` — Add path-overlap tracking and conflict check to `run_ready_queue_direct()`

## Details

### Track running paths

Add a `HashSet<String>` called `running_paths` that accumulates the `paths` from all currently-running beans. When a bean finishes, remove its paths from the set.

### Add conflict check

Before starting a ready bean, check if any of its `paths` overlap with `running_paths`. If so, skip it this round — it'll be reconsidered next time a bean finishes.

```rust
/// Check if a bean's paths conflict with currently-running beans.
fn has_path_conflict(bean: &SizedBean, running_paths: &HashSet<String>) -> bool {
    bean.paths.iter().any(|p| running_paths.contains(p))
}
```

### Modify `run_ready_queue_direct()` flow

In the main loop where ready beans are selected and started:

1. After computing `ready_beans`, filter out those with path conflicts
2. When starting a bean, add its `paths` to `running_paths`
3. When a bean completes (success or failure), remove its `paths` from `running_paths`
4. Beans with empty `paths` never conflict — always schedulable

Key: this is purely a scheduling optimization. It doesn't replace file locking (which protects against writes). It prevents the situation entirely by not scheduling conflicting work.

### Edge cases

- **Empty paths**: Bean has no `paths` → never conflicts, always eligible
- **Multiple conflicts**: Bean A conflicts with both B and C — must wait for both to finish
- **All ready beans conflict**: Only start one at a time (degrades to serial for that file set)
- **No paths on any bean**: Feature is effectively a no-op, zero overhead

let ready_ids: Vec<String> = remaining
    .values()
    .filter(|b| is_bean_ready(b, &completed, &all_bean_ids, all_beans))
    .map(|b| b.id.clone())
    .collect();

// Sort ready beans by priority then ID (stable ordering)

    .iter()
    .filter_map(|id| remaining.get(id).cloned())
    .collect();

    a.priority
        .cmp(&b.priority)
        .then_with(|| natural_cmp(&a.id, &b.id))
});

    if running_count >= max_jobs {
        break;

    remaining.remove(&sb.id);
    running_count += 1;
    // ... spawn thread ...
