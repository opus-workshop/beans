## Problem

`bn run` spawns child agent processes. On Ctrl+C, there's no signal handler — children may become orphans or leave beans in a claimed state.

## Approach

Install a signal handler that:
1. Forwards SIGTERM to all child processes
2. Waits briefly for graceful shutdown
3. SIGKILL any remaining children
4. Releases claims on beans that were being worked on

### Files to modify
- `src/commands/run.rs` — install signal handler before wave execution
- `src/spawner.rs` — add `shutdown_all()` method

### Design
- Use `libc` (already a dep) for signal handling, or `ctrlc` crate
- Set an AtomicBool flag on SIGINT/SIGTERM
- Check flag in the poll loop between waves
- On shutdown: kill children, release claims, print summary

### Acceptance Criteria
- [ ] Signal handling code in run.rs
- [ ] Ctrl+C during bn run kills child agents
- [ ] Claims released on interrupted beans
- [ ] All existing run tests still pass
