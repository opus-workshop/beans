New command: `bn run [OPTIONS] [ID]`

Dispatch ready beans to agents. Finds ready beans, sizes them, spawns agents
for small beans (implement) and optionally large beans (plan).

## CLI Definition (in src/cli.rs)

```rust
Run {
    /// Bean ID. Without ID, processes all ready beans.
    id: Option<String>,

    /// Max parallel agents
    #[arg(short = 'j', long, default_value = "4")]
    jobs: u32,

    /// Show plan without spawning
    #[arg(long)]
    dry_run: bool,

    /// Keep running until no ready beans remain
    #[arg(long, name = "loop")]
    loop_mode: bool,

    /// Also plan large beans autonomously
    #[arg(long)]
    auto_plan: bool,

    /// Continue past failures
    #[arg(long)]
    keep_going: bool,

    /// Max time per agent in minutes
    #[arg(long, default_value = "30")]
    timeout: u32,

    /// Kill agent if no output for N minutes
    #[arg(long, default_value = "5")]
    idle_timeout: u32,

    /// Watch mode flags (--watch, --foreground, --stop) handled in 66.9
},
```

## Implementation (src/commands/run.rs)

`cmd_run()` flow:
1. Load config (with extends)
2. Validate run template exists (bail with setup hint if not)
3. If --stop: stop daemon and exit
4. If ID provided: filter to just that bean (or its children if parent)
5. Call orchestrator::plan_dispatch() to get waves + skipped
6. If --dry-run: print plan and exit
7. Report skipped beans ("N beans need planning, run bn plan")
8. For each wave:
   a. Spawn agents up to max_concurrent (via Spawner)
   b. Poll for completion (sleep + check_completed loop)
   c. If --keep-going: continue even if some fail
   d. If not --keep-going: stop on first failure in wave
9. If --loop: go back to step 5 (new beans may be ready)
10. Print summary: N implemented, N planned, N failed, N skipped

```
  ✓ 5.3  Add config parsing       implement  0:45

```

## Error cases
- No run template configured → "No agent configured. Run bn init --setup"
- No ready beans → "No ready beans. Use bn status to see what's going on."
- Bean ID not found → standard bn error
- Agent spawn failure → log error, mark bean failed, continue if --keep-going
