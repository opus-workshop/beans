New module src/spawner.rs — agent-agnostic process spawning, tracking, and log capture.

## Core types

```rust
pub struct AgentProcess {
    pub bean_id: String,
    pub bean_title: String,
    pub action: AgentAction,     // Implement or Plan
    pub pid: u32,
    pub started_at: Instant,
    pub log_path: PathBuf,
}

pub enum AgentAction {
    Implement,
    Plan,
}

pub struct Spawner {
    running: HashMap<String, AgentProcess>,
}
```

## Functions

1. `Spawner::new()` — create empty spawner
2. `Spawner::spawn(bean_id, action, config) -> Result<()>`
   - Build command from config.run or config.plan template
   - Replace `{id}` with bean_id
   - Redirect stdout/stderr to log file
   - Spawn via `sh -c <cmd>`
   - Track in running map
3. `Spawner::check_completed() -> Vec<CompletedAgent>`
   - Non-blocking check via try_wait()
   - Remove completed from running map
   - Return results with success/failure
4. `Spawner::running_count() -> usize`
5. `Spawner::can_spawn(max_concurrent) -> bool`
6. `Spawner::list_running() -> Vec<&AgentProcess>`
7. `Spawner::kill_all()`

## Log management
- Logs go to ~/.local/share/beans/logs/{bean_id}-{timestamp}.log
- `log_dir()` — return/create log directory
- `find_latest_log(bean_id)` — find most recent log for a bean
- `find_all_logs(bean_id)` — find all logs for a bean

## Bean lifecycle integration
- Before spawning: `bn claim {id}` (via Command::new("bn"))
- After failure: `bn claim {id} --release`
- Template is responsible for running `bn close` on success
