Add --watch, --foreground, and --stop flags to bn run.
This replaces bw daemon functionality.

## New dependencies in Cargo.toml
- notify = "6" (file watching)
- signal-hook = "0.3" (signal handling)
- libc = "0.2" (PID checking)
- daemonize = "0.5" (background mode)

## New module: src/daemon.rs

### Daemon lifecycle
- `start_daemon(config, foreground)` — PID file, signal handling, main loop
- `stop_daemon()` — read PID file, send SIGTERM, wait, cleanup
- `is_daemon_running()` — check PID file and process
- PID file: ~/.local/share/beans/daemon.pid

### Main loop (same as bw)
1. Set up file watcher on .beans/ directory (notify crate)
2. On file change or poll_interval timeout:
   a. Run poll cycle: get ready beans, size, spawn
   b. Check completed agents
3. On SIGINT/SIGTERM: kill all agents, cleanup, exit

### Integration with bn run
In src/commands/run.rs, add handling for:
- --watch: call start_daemon (background by default)
- --watch --foreground: call start_daemon(foreground=true)
- --stop: call stop_daemon
