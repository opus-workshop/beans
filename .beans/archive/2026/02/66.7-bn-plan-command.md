New command: `bn plan [OPTIONS] [ID]`

Interactively plan a large bean into children. Without an ID, picks the
highest-priority ready bean that exceeds max_tokens.

## CLI Definition (in src/cli.rs)

```rust
Plan {
    /// Bean ID to plan. Omit to pick automatically.
    id: Option<String>,

    /// Suggest a split strategy
    #[arg(long)]
    strategy: Option<String>,  // feature, layer, phase, file

    /// Non-interactive autonomous planning
    #[arg(long)]
    auto: bool,

    /// Plan even if bean is below max_tokens
    #[arg(long)]
    force: bool,

    /// Show proposed split without creating
    #[arg(long)]
    dry_run: bool,
},
```

## Implementation (src/commands/plan.rs)

`cmd_plan()` flow:

1. Load config (with extends)
2. If no ID: find all ready beans, size them, filter to above max_tokens,
   sort by priority then ID, pick first. If none found, print helpful message.
3. If ID provided but bean is below max_tokens (and no --force):
   print "Bean is Nk tokens — small enough to run directly. Use bn run {id}"
4. Validate plan template exists (bail with setup hint if not)
5. If --auto: spawn agent with plan template, wait for completion, exit
6. If interactive (default): spawn agent with plan template.
   The agent handles the interactive session — bn plan just launches it.
7. Print result: "Planning complete. Use bn tree {id} to see children."
