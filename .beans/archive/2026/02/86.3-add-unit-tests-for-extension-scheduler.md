## Task

Add unit tests for `~/.pi/agent/extensions/beans/scheduler.ts` — the ready-queue scheduler that computes dependency order.

## Why this matters

The scheduler decides what order beans execute in. If it's wrong, agents run out of order and stomp on each other. This is silent corruption — you won't know until something breaks downstream.

## What to test

Read `scheduler.ts` first to understand the interface, then test:

1. **No dependencies** — all beans ready immediately, single wave
2. **Linear chain** — A produces X, B requires X → A before B
3. **Diamond** — A produces X, B produces Y, C requires X+Y → C waits for both
4. **Independent groups** — beans with no relationship run in parallel
5. **Priority ordering** — within a wave, lower priority number runs first
6. **Already-closed beans** — closed beans' produces are pre-satisfied
7. **Circular dependency** — should not infinite loop (error or break cycle)
8. **Empty input** — no beans → no waves

## Approach

Use a simple test runner (just assertions + try/catch, no framework needed). Import the scheduler functions directly. Write the test file as `scheduler.test.ts` alongside the source.

## Acceptance Criteria
- scheduler.test.ts exists
- All tests pass when run with `npx tsx scheduler.test.ts`
- Covers at least the 8 scenarios above
