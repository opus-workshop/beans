## Task
Make `bn plan --auto` a self-contained autonomous decomposition pipeline.
Instead of relying on the auto skill, bn plan builds a rich decomposition
prompt and spawns a pi agent with it directly.

## Changes

### 1. Build decomposition prompt in bn plan
When `bn plan <id> --auto` runs, bn should:
1. Read the bean (bn show <id>)
2. Calculate token size
3. Build a decomposition prompt that includes:
   - The bean's full description
   - Sizing rules (≤5 functions → atomic, >5 → split)
   - Split strategy guidance (by-feature, by-layer, by-file)
   - Context embedding rules (always embed types, patterns, signatures)
   - bn create syntax with --parent, --produces, --requires, --verify
   - Examples of good child bean descriptions
4. Spawn pi with this prompt directly (not via template)

### 2. Embed the auto skill's core wisdom
The key rules from the auto skill to bake in:
- Count functions concretely, don't estimate
- Maximize parallelism — prefer independent beans over chains
- Embed context into descriptions, don't reference it
- Every child needs a verify command
- Be specific: file paths with actions, concrete steps, edge cases
- 2-4 children for medium beans, 3-5 for large

### 3. Keep template fallback
If config.plan is set, use the template (backward compat).
If not set, use the built-in decomposition prompt.
