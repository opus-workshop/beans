Two new commands for monitoring agent activity.

## bn agents

Show running and recently completed agents.

```rust
Agents {
    #[arg(long)]
    json: bool,
},
```

Implementation:
- Read agent state from persistence file (~/.local/share/beans/agents.json)
- For running: check if PID is still alive, show elapsed time
- For completed: show exit status, duration
- Clean up stale entries (PID no longer running)

Output format:
```
BEAN   TITLE                    ACTION      PID    ELAPSED
5.1    Define user types        implement   42310  1m 32s
5.2    Implement handlers       implement   42311  0m 48s
8      Auth system              plan        42315  0m 12s
```

JSON output: array of objects with bean_id, title, action, pid, elapsed_secs, status.

If no agents: "No running agents."

## bn logs

View agent output from log files.

```rust
Logs {
    /// Bean ID
    id: String,

    /// Follow output (tail -f)
    #[arg(short, long)]
    follow: bool,

    /// Show all runs, not just latest
    #[arg(long)]
    all: bool,
},
```

Implementation:
- find_latest_log(bean_id) from spawner module
- Default: cat the latest log file
- --follow: tail -f the log file (exec into tail)
- --all: list all log files with timestamps, cat each with header

Error cases:
- No logs found: "No logs for bean {id}. Has it been dispatched with bn run?"

## Persistence format

~/.local/share/beans/agents.json:
```json
{
  "5.1": { "pid": 42310, "title": "...", "action": "implement",
           "started_at": 1708000000, "log_path": "..." },
  "5.2": { ... }
}
```

The spawner writes to this file on spawn/completion.
bn agents reads it. Simple file-based IPC.
