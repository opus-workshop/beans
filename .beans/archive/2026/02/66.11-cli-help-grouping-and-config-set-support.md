Two tasks:

## 1. Group commands in --help output

Use clap's help_heading attribute to group commands into sections:

```
Commands:
  TASKS
  init, create, quick, show, list, edit, update, claim, close, verify, reopen, delete

  QUERY
  status, ready, blocked, tree, graph, context

  AGENTS
  run, plan, agents, logs

  DEPENDENCIES
  dep, adopt

  MAINTENANCE
  tidy, sync, doctor, stats, config, trust, unarchive, resolve
```

In src/cli.rs, add `#[command(help_heading = "AGENTS")]` etc. to each variant.
Or use `next_help_heading` in the derive macro.

## 2. Add new config keys to bn config set/get

In src/commands/config_cmd.rs, add support for:
- `bn config set plan "<template>"` 
- `bn config set max_concurrent <n>`
- `bn config set poll_interval <n>`
- `bn config get plan`
- `bn config get max_concurrent`
- `bn config get poll_interval`

## Files
- src/cli.rs (modify — add help_heading attributes)
- src/commands/config_cmd.rs (modify — add new keys)

## Context

### Current config_cmd.rs pattern:
The existing cmd_config_set handles keys like "run", "max_tokens" etc.
Add the three new keys following the same pattern.

## Tests
- bn --help shows AGENTS heading
- bn config set plan "test" / bn config get plan round-trips
- bn config set max_concurrent 8 / get returns 8
- bn config set poll_interval 60 / get returns 60
