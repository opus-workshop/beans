Enhance bn init to guide users through agent setup.

## New CLI flags

```rust
Init {
    name: Option<String>,

    /// Use a known agent preset (pi, claude, aider)
    #[arg(long)]
    agent: Option<String>,

    /// Custom run command template
    #[arg(long)]
    run: Option<String>,

    /// Custom plan command template
    #[arg(long)]
    plan: Option<String>,

    /// Reconfigure agent on existing project
    #[arg(long)]
    setup: bool,

    /// Skip agent setup
    #[arg(long)]
    no_agent: bool,
},
```

## Flow

### Fresh init (no .beans/ exists):
1. Create .beans/ directory (existing logic)
2. If --no-agent: save config without run/plan, done
3. If --agent <name>: apply preset, save config, verify agent, done
4. If --run/--plan provided: use those directly, done
5. If interactive (TTY):
   a. detect_agents() — scan for installed CLIs
   b. Print found/not-found status for each
   c. Prompt user to pick (or enter custom command)
   d. Apply selected preset to config
   e. Verify agent (run version command)
   f. Print success + next steps
6. If not interactive (no TTY, piped):
   Save config without run/plan, print hint to configure

### Re-init (--setup or .beans/ already exists):
- If --setup: run agent setup flow (steps 3-6 above)
- If not --setup: print current config summary, suggest --setup

## Interactive prompt (when TTY)
Use simple stdin-based selection (no TUI library needed):
```
Agent setup
  Checking for agent CLIs...
  ✓ pi found (/usr/local/bin/pi)
  ✗ claude not found
  ✗ aider not found

Which agent? [1] pi  [2] claude  [3] aider  [4] custom
>
```

## Verification
After setting the agent, run its version command:
```
Verifying agent...
  ✓ pi --version → pi 0.9.0
```
If version command fails, warn but don't block.
