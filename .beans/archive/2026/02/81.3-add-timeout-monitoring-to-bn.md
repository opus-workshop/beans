## Task
1. Create src/timeout.rs with timeout monitoring for spawned agent processes
2. Add `pub mod timeout;` to src/lib.rs

Port the timeout system from deli/src/timeout.rs. This monitors a child process's
stdout and kills it if total time exceeds a limit or if no output is produced
within the idle timeout window.

## Files
- src/timeout.rs (create)

## Context

### Core types
```rust
#[derive(Debug, Clone, Default)]
pub struct TimeoutConfig {
    pub total_timeout: Duration,
    pub idle_timeout: Duration,
}

#[derive(Debug, PartialEq)]
pub enum MonitorResult {
    Completed,
    TotalTimeout,
    IdleTimeout,
    Killed,
}
```

### monitor_process function
Takes a Child process + stdout handle + TimeoutConfig + line callback.
Runs in the calling thread (blocking):
1. Read stdout line by line via BufReader
2. On each line, reset idle timer, call the on_line callback
3. If total time exceeded → kill process → return TotalTimeout
4. If idle time exceeded → kill process → return IdleTimeout
5. If process exits naturally → return Completed

The callback `on_line: impl FnMut(&str)` is called for every line of output,
enabling log capture and event parsing simultaneously.

### Key detail from deli
The monitor uses `child.stdout` with BufReader in a loop. It checks elapsed
time after each line. For idle timeout, it tracks the timestamp of the last
line received.
