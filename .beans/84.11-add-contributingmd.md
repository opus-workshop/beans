## Problem

No contributor guide. Someone wanting to add a command has to reverse-engineer the pattern from existing code.

## Content

```markdown
# Contributing to beans

## Development Setup
cargo build && cargo test

## Adding a Command
1. Add variant to `Command` enum in `src/cli.rs`
2. Create `src/commands/yourcommand.rs` with `pub fn cmd_yourcommand(beans_dir: &Path, ...) -> Result<()>`
3. Add `pub mod yourcommand;` in `src/commands/mod.rs`
4. Add match arm in `src/main.rs` dispatch
5. Add tests as `#[cfg(test)] mod tests` in your command file

## Conventions
- Error handling: anyhow::Result + .context()
- Testing: inline #[cfg(test)] modules
- IDs: always validate via util::validate_bean_id()
- Sorting: use util::natural_cmp for bean IDs

## Running Tests
cargo test                    # all
cargo test commands::close    # specific module
```

### Acceptance Criteria
- [ ] CONTRIBUTING.md exists at project root
- [ ] Documents how to add a command (the 4-file pattern)
