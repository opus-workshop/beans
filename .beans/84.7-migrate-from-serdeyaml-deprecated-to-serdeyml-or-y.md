## Problem

`serde_yaml 0.9` is deprecated (marked `+deprecated` in cargo tree output). Continuing to use it means no security patches or bug fixes.

## Approach

Migrate to `serde_yml` (community fork, drop-in replacement) or `serde_yaml_ng`.

### Files to modify
- `Cargo.toml` — replace `serde_yaml` with chosen replacement
- `src/**/*.rs` — find/replace `serde_yaml::` → new crate name

### Investigation needed
- Check which fork is most active/maintained
- `serde_yml` appears to be the most popular drop-in replacement
- API should be identical — just a crate name change

### Acceptance Criteria
- [ ] serde_yaml removed from Cargo.toml
- [ ] Replacement crate added
- [ ] All YAML serialization/deserialization works identically
- [ ] All 851+ tests pass
- [ ] No deprecation warnings in cargo build
