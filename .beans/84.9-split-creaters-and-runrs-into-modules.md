## Problem

`create.rs` (1,957L) and `run.rs` (1,663L) are oversized. They're hard to navigate and have high churn.

## Approach

Same pattern as close.rs split — convert to directory modules.

### create.rs → create/
```
src/commands/create/
├── mod.rs      — cmd_create orchestration
├── slug.rs     — title-to-slug, ID generation
├── template.rs — interactive creation, prefill
└── validate.rs — field validation, duplicate detection
```

### run.rs → run/ (or run_cmd/)
```
src/commands/run_cmd/
├── mod.rs      — cmd_run, run_once, run_loop
├── plan.rs     — plan_dispatch, compute_waves, sizing
├── template.rs — SpawnMode::Template execution
└── direct.rs   — SpawnMode::Direct (pi) execution
```

### Acceptance Criteria
- [ ] Both files converted to directory modules
- [ ] No file over 1000 lines
- [ ] All existing tests pass
- [ ] No public API changes
